# 涨停预测数据集构建优化总结 - V3.1版本

## 🎯 核心突破总结

### 1. 标签重构：从"状态"到"事件"

**V3.0版本问题：**
- ❌ 标签定义：第11个tick是否处于涨停状态
- ❌ 假正样本：单日产生~20万个假正样本（涨停板上的每秒都是正样本）
- ❌ 业务逻辑：无法在已封死时买入，预测意义不大

**V3.1版本改进：**
- ✅ 标签定义：价格是否从非涨停突破到涨停（触板瞬间）
- ✅ 标签逻辑：`label = (current[T11] >= limit_price AND current[T10] < limit_price)`
- ✅ 正样本数：单日从20万骤降至10-50个
- ✅ 业务逻辑：完全符合真实交易规律

### 2. 内存优化：从"先平铺后过滤"到"先过滤后平铺"

**V3.0版本问题：**
- ❌ 处理流程：全量数据 → 165维平铺 → 欠采样
- ❌ 内存占用：1335万tick × 165列 ≈ 20GB
- ❌ 数据浪费：95%的tick数据对预测无价值

**V3.1版本改进：**
- ✅ 处理流程：事件标签 → 前置过滤 → 按需平铺 → 欠采样
- ✅ 内存占用：1-2万样本 × 165列 ≈ 1-2GB
- ✅ 内存节省：90%+
- ✅ 处理速度：提升5-10倍（避免全表shift操作）

### 3. 前置内存阻断策略

**保留条件（高价值样本）：**
1. 正样本：触板瞬间（label == 1）
2. 困难负样本：距离涨停<2%但最终未封板的时刻

**剔除条件（无用数据）：**
1. 死板状态：已经在涨停板上（current >= limit_price）
2. 垃圾时间：距离涨停≥2%的时刻（无预测价值）
3. 边界数据：前9个tick无法构筑窗口，最后一个tick无未来标签

## 🔧 技术实现要点

### 1. 事件驱动标签实现

```python
def generate_event_driven_label(df):
    # 获取T11（下一秒）状态
    next_price = df['current'].shift(-1)

    # 获取T10（当前秒）状态
    current_price = df['current']
    limit_price = df['limit_price']

    # 核心逻辑：检测价格是否从非涨停突破到涨停
    next_is_limit = next_price >= limit_price
    current_is_limit = current_price >= limit_price

    # 事件驱动标签：仅保留突破瞬间
    df['label'] = (next_is_limit & ~current_is_limit).astype(int)

    # 边界处理：最后一个tick无法检测未来状态
    return df.iloc[:-1].copy()
```

### 2. 前置内存阻断实现

```python
def filter_event_samples(df, limit_dist_threshold=0.02):
    # 第一步：生成事件驱动标签
    df = generate_event_driven_label(df)

    # 第二步：计算距离涨停价
    df['dist_to_limit'] = (df['limit_price'] - df['current']) / df['limit_price']

    # 第三步：构建内存过滤掩码
    condition_a = df['label'] == 1  # 正样本（触板瞬间）
    condition_b = df['dist_to_limit'] < limit_dist_threshold  # 困难负样本
    is_dead_limit = df['current'] >= df['limit_price']  # 死板状态

    # 最终保留条件
    keep_mask = (condition_a | condition_b) & (~is_dead_limit)

    # 边界处理：前9个tick无法构筑窗口，最后一个tick无未来标签
    if len(df) > 10:
        keep_mask.iloc[:9] = False
        keep_mask.iloc[-1] = False

    return df, df[keep_mask].index
```

### 3. 按需特征平铺实现

```python
def _build_window_features_on_demand(self, df, valid_indices):
    # 核心思路：仅对valid_indices进行特征平铺
    result_df = df.loc[valid_indices, ['code', 'time', 'label'] + self.BASE_FEATURES].copy()

    # 重命名T10特征（窗口末端，用于采样）
    for col in self.BASE_FEATURES:
        result_df[f'{col}_last'] = result_df[col].values
        del result_df[col]

    # 获取位置索引
    loc_indices = df.index.get_indexer(valid_indices)

    # 回溯提取T9至T1的特征
    for lag in range(1, self.window_size):
        tick_name = f'T{self.window_size - lag}'
        lag_locs = loc_indices - lag

        # 提取历史数据
        for col in self.BASE_FEATURES:
            past_values = df.iloc[lag_locs][col].values
            result_df[f'{col}_{tick_name}'] = past_values

    return result_df
```
