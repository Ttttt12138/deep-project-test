## 问题概述
根据分析，"有触板事件(142个)但被过滤"的根本原因是：
- V3.2逻辑本身正确，但真实数据处理流程存在多个失败点
- 95%失败率（无触板事件且无困难负样本）
- 142只有触板但被过滤

## 根本原因分析

### 1. 基准价格获取失败 (最常见)
**问题位置**: `training_set_builder.py` - `_get_benchmark_price()`

**问题描述**:
```python
def _get_benchmark_price(self, csv_file: Path) -> float:
    """获取基准价格"""
    try:
        df_raw = pd.read_csv(csv_file, nrows=100)
        # 优先级：买一价 > 卖一价 > 开盘价 > 当前价
        for price_col in ['b1_p', 'a1_p', 'open', 'current']:
            if price_col in df_raw.columns:
                valid_prices = df_raw[df_raw[price_col] > 0][price_col]
                if len(valid_prices) > 0:
                    return valid_prices.iloc[0]
    except Exception:
        pass
    return None
```

**失败原因**:
- 很多股票前100行数据可能都在集合竞价时间（<09:30）
- 这些时间的价格通常为0或无效
- 导致所有候选价格列都无效

**影响**: 基准价格为None → 股票被直接跳过

### 2. 特征提取失败
**问题位置**: `limit_up_processor.py` - `extract_limit_up_features()`

**问题描述**:
- 15个核心特征需要从原始列计算
- 如果原始列缺失或数据质量差，特征提取失败
- 窗口构建器检查必需特征，缺失则返回空DataFrame

### 3. 窗口构建边界问题
**问题位置**: `event_window_builder.py` - `filter_and_label_events()`

**问题描述**:
```python
# 边界处理：只过滤既不是正样本也不是困难负样本的最后一个tick
if len(df) > 0:
    last_tick_is_important = (df['label'].iloc[-1] == 1) or (df['dist_to_limit'].iloc[-1] < limit_dist_threshold)
    if not last_tick_is_important:
        keep_mask.iloc[-1] = False
```

**问题**:
- 触板事件发生在数据末尾时，可能被边界过滤误伤
- 即使触板被检测到，窗口构建可能因为历史数据不足而失败

### 4. 数据过滤过于严格
**问题位置**: `limit_up_processor.py` - `filter_invalid_ticks()`

**问题描述**:
- 8条清理规则可能过滤掉有价值的触板数据
- 特别是规则3（集合竞价）和规则8（volume为0）

## 修复方案

### 方案1: 增强基准价格获取逻辑（优先级最高）

```python
def _get_benchmark_price_robust(self, csv_file: Path) -> float:
    """
    增强版基准价格获取 - 更容错

    改进:
    1. 读取更多行（nrows=1000）
    2. 跳过集合竞价时间
    3. 使用中位数代替首值
    4. 最后防线：基于涨停价反推
    """
    try:
        df_raw = pd.read_csv(csv_file, nrows=1000)

        # 转换时间列
        if 'time' in df_raw.columns:
            df_raw['time'] = pd.to_datetime(df_raw['time'].astype(str), format='%Y%m%d%H%M%S', errors='coerce')
            # 过滤集合竞价时间（>=09:30）
            df_raw = df_raw[df_raw['time'].dt.time >= pd.to_datetime('09:30:00').time()]

        # 优先级：买一价 > 卖一价 > 开盘价 > 当前价
        for price_col in ['b1_p', 'a1_p', 'open', 'current']:
            if price_col in df_raw.columns:
                valid_prices = df_raw[df_raw[price_col] > 0][price_col]

                if len(valid_prices) > 0:
                    # 使用中位数，避免异常值影响
                    return valid_prices.median()

        # 最后防线：如果有limit_price列，尝试反推
        if 'limit_price' in df_raw.columns and 'current' in df_raw.columns:
            valid_limit = df_raw[df_raw['limit_price'] > 0]['limit_price']
            if len(valid_limit) > 0:
                # 假设10%涨停比例，反推preclose
                limit_price_median = valid_limit.median()
                return round(limit_price_median / 1.1, 2)

    except Exception as e:
        pass

    return None
```

    required_features = [

        'ret_1tick', 'vol_delta', 'money_delta'
    ]

    try:
        # 原有特征提取逻辑
        # ...
    except Exception as e:
        # 容错处理：至少返回dist_to_limit和基础列
        if 'current' in df.columns and 'limit_price' in df.columns:
            df['dist_to_limit'] = (df['limit_price'] - df['current']) / df['limit_price']
            df['ticks_to_limit'] = (df['limit_price'] - df['current']) / tick_size
