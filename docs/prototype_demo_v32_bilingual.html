🎯 建模目标

🎯 Modeling Goal

给定连续 10个Tick 的窗口信息，预测第 11个Tick 是否会发生首次触板事件

Given 10 consecutive Ticks, predict whether a first limit-up touch will occur at the 11th Tick

📊 业务逻辑

📊 Business Logic

完全对齐"今天能不能上板"的预测目标，首次触板最具预测价值

Perfectly aligns with "can it limit-up today" objective, first touch has most predictive value

V3.2 核心特性

V3.2 Core Features

1
首次触板检测 First Touch Detection

按股票分组，只保留每只股票的首次触板，解决多次触板问题

Group by stock, keep only first touch per stock, solves multi-touch problem

2
智能NaN填充 Smart NaN Filling

价格类特征用向前/向后平推，成交量类用0填充，避免未来函数泄露

Price features: ffill/bfill, volume features: fill with 0, prevents future function leakage

3
前置内存阻断 Early Memory Blocking

过滤95%无用数据，仅保留正样本和困难负样本（距离<5%）

Filter 95% useless data, keep only positives and hard negatives (distance<5%)

4
按需特征平铺 On-demand Feature Flattening

仅对1-2万高价值样本进行165维提取，避免全表shift操作

165D extraction only on 10-20K high-value samples, avoids full-table shift

数据处理流程

Data Processing Pipeline

步骤 1：首次触板标签生成 Step 1: First Touch Label Generation

检测所有触板瞬间 → 按股票分组 → 仅保留首次触板

Detect all touch moments → group by stock → keep only first touch

步骤 2：前置内存阻断 Step 2: Early Memory Blocking

保留正样本 + 困难负样本（dist<5%），剔除死板和垃圾时间

Keep positives + hard negatives (dist<5%), exclude dead limit and garbage time

步骤 3：按需特征平铺 Step 3: On-demand Feature Flattening

15个核心特征 × 10个时间位置 + 15个末端特征 = 165维

15 core features × 10 time positions + 15 final features = 165D

步骤 4：智能NaN填充 Step 4: Smart NaN Filling

价格类：ffill/bfill，成交量类：0填充，严禁mean()

Price: ffill/bfill, volume: fill with 0, strictly no mean()

步骤 5：两层负采样 Step 5: Two-layer Negative Sampling

基于距离的分层欠采样，按交易日独立处理

Distance-based stratified undersampling, process each trading day independently
