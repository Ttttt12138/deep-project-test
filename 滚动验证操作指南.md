# 滚动验证操作指南

## 系统概述

采用 Expanding Window 策略，按月滚动验证：
- Fold 1: 训练 Jan~Mar, 验证 Apr
- Fold 2: 训练 Jan~Apr, 验证 May
- ...
- Fold 9: 训练 Jan~Nov, 验证 Dec

## 快速开始

### Step 1: 合并数据

```bash
python scripts/merge_daily_data.py
```

将 `data/daily_train_undersampled/{01-12}/` 下的 CSV 合并为 `data/merged/multi_day_train.csv`

### Step 2: 运行滚动验证

```bash
python scripts/rolling_cv.py
```

自动完成：加载数据 → 生成Fold → 逐月训练 → 评估 → 保存结果

### Step 3: 生成可视化图表

```bash
python scripts/rolling_cv_plot.py
```

生成 `data/rolling_cv/rolling_cv_plot.png`

## 评估指标

| 指标 | 参考值 | 说明 |
|------|--------|------|
| ROC-AUC | > 0.75 可用, > 0.85 良好 | 整体排序能力 |
| PR-AUC | 越高越好 | 不平衡数据更真实 |
| Precision | > 50% 有实用价值 | 高精度比高召回更有价值 |
| Recall | 视策略决定 | 触板事件召回率 |
| AUC方差 | std < 0.1 稳定 | 跨Fold稳定性 |

**指标优先级**：PR-AUC > Precision@0.5 > ROC-AUC > AUC方差

---

## 结果解读

### 正常表现
- ROC-AUC > 0.99, PR-AUC > 0.95 → 优秀
- 各Fold方差 < 0.01 → 稳定

### 异常诊断

| 现象 | 说明 | 建议 |
|------|------|------|
| AUC逐月下滑 | 市场风格漂移 | 改用Sliding Window |
| AUC波动±0.1+ | 验证集正样本太少 | 扩大验证窗口 |
| PR-AUC很低 | 正样本预测效果差 | 检查scale_pos_weight |
| dist_to_limit占比>50% | 疑似数据泄露 | 检查特征边界 |

## 特征重要性检查

```bash
python scripts/rolling_cv.py && python -c "
import joblib, pandas as pd

model = joblib.load('models/rolling_cv/fold1_04.pkl')
df = pd.read_csv('data/merged/multi_day_train.csv')
cols = [c for c in df.columns if c not in ['label','date','month','code','time','window_start_time','window_end_time']]
imp = pd.DataFrame({'feature': cols, 'importance': model.feature_importances_})
imp = imp.sort_values('importance', ascending=False)

# 检查dist_to_limit占比
dist_cols = [c for c in cols if 'dist_to_limit' in c]
dist_imp = imp[imp['feature'].isin(dist_cols)]['importance'].sum()
total_imp = imp['importance'].sum()
print(f'dist_to_limit占比: {dist_imp/total_imp*100:.2f}%')
print()
print('Top 20 特征:')
print(imp.head(20).to_string(index=False))
"
```

**检查标准**：dist_to_limit 占比应 < 50%，否则存在数据泄露风险

## 模型配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| num_leaves | 63 | 控制模型复杂度 |
| learning_rate | 0.05 | 学习率 |
| n_estimators | 500 | 最大树数量 |
| early_stopping | 50 | 早停轮数 |
| scale_pos_weight | 动态 | 负样本/正样本 |
| feature_fraction | 0.8 | 每棵树随机特征比例 |

---

## 注意事项

1. **严禁数据泄露**：特征构建只能使用 T0~T9 的信息，T10 的价格仅用于生成标签
2. **正样本稀缺**：单日约 10~50 个正样本，若训练集正样本 < 200，模型可能无法收敛
3. **不要过度调参**：先跑通baseline，再根据诊断结果迭代优化
