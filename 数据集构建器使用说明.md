# 数据集构建器使用说明（含欠采样功能）

## 功能概述

`scripts/dataset_builder.py` 已经完整集成了负样本欠采样功能，可以在数据集构建流程中自动执行两层欠采样策略。

## 核心特性

1. **两步处理策略**：解压 → 处理并自动清理
2. **负样本欠采样**：基于 `dist_to_limit` 分层 + 比例控制
3. **灵活配置**：支持强制采样和自动检测模式
4. **完整输出**：同时生成完整数据集和采样后的划分数据集

## 使用方法

### 1. 基本使用（默认启用欠采样）

```bash
python scripts/dataset_builder.py \
    --input "2025/1/2025-01-02.7z" \
    --output "data" \
    --split
```

**效果**：
- 生成完整数据集：`data/2025-01-02_dataset.csv`
- 生成划分数据集：`data/split_datasets/2025-01-02/`
  - `train.csv`（已欠采样）
  - `validation.csv`（保持原样）
  - `test.csv`（保持原样）

### 2. 禁用欠采样

```bash
python scripts/dataset_builder.py \
    --input "2025/1/2025-01-02.7z" \
    --output "data" \
    --split \
    --disable-undersampling
```

### 3. 自动检测模式

```bash
python scripts/dataset_builder.py \
    --input "2025/1/2025-01-02.7z" \
    --output "data" \
    --split \
    --auto-undersample
```

**效果**：当训练集正负比例 > 1:8 时自动执行欠采样

### 4. 自定义欠采样参数

```bash
python scripts/dataset_builder.py \
    --input "2025/1/2025-01-02.7z" \
    --output "data" \
    --split \
    --sampling-target-ratio 3.0 \
    --sampling-auto-threshold 10.0
```

## 参数说明

### 基本参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--input` | 必填 | 7z文件路径 |
| `--output` | `data` | 输出目录 |
| `--split` | False | 是否划分数据集 |
| `--train-ratio` | 0.70 | 训练集比例 |
| `--val-ratio` | 0.15 | 验证集比例 |

### 欠采样参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--enable-undersampling` | True | 启用欠采样 |
| `--disable-undersampling` | False | 禁用欠采样 |
| `--auto-undersample` | False | 自动检测模式 |
| `--sampling-target-ratio` | 5.0 | 目标正负比例（1:N中的N） |
| `--sampling-auto-threshold` | 8.0 | 自动检测阈值 |

## 欠采样策略详情

### 两层采样策略

#### 第一层：分层筛选
基于 `dist_to_limit` 分层：

- **接近涨停**（≤1%）：保留 **100%**
- **中等距离**（1%-5%）：保留 **30%**
- **远离涨停**（>5%）：保留 **5%**

#### 第二层：比例控制
随机采样到目标比例 **1:5**（默认）

### 重要规则

✅ **仅训练集欠采样**：验证集和测试集保持原始分布
✅ **正样本100%保留**：确保所有涨停信号都被学习
✅ **固定随机种子**：使用随机种子42确保可重复性

## 使用建议

### 推荐使用场景

1. **首次构建数据集**：使用默认参数
2. **调试模型**：使用 `--auto-undersample` 灵活调整
3. **对比实验**：分别生成有/无欠采样的数据集

### 性能优化

1. **数据量大时**：使用 `--enable-undersampling` 显著减少训练时间
2. **GPU内存受限**：欠采样后可使用更大的batch size
3. **快速迭代**：使用 `--sampling-target-ratio 3.0` 进一步减少样本数

### 注意事项

⚠️ **不要对验证集/测试集采样**：这会导致评估结果不准确
⚠️ **保持随机种子一致**：确保实验可重复
⚠️ **监控采样效果**：检查正样本是否100%保留

## 故障排除

### 问题1：欠采样后正样本数量减少

**原因**：正样本应该100%保留，如果减少可能有问题
**解决**：检查采样参数，确保 `dist_to_limit` 和 `label` 列存在

### 问题2：采样比例不符合预期

**原因**：负样本数量不足以达到目标比例
**解决**：调整 `sampling_target_ratio` 或 `sampling_auto_threshold`

### 问题3：内存不足

**原因**：完整数据集太大
**解决**：使用 `--enable-undersampling` 或分批处理
