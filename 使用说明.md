# 涨停预测系统使用说明

## 系统概述

这是一个基于Tick数据的股票涨停预测系统，使用LightGBM分类器预测下一个tick是否涨停。

## 快速开始

### 一键运行完整流程

```bash
python run_system.py --mode quick
```

这将自动执行以下步骤：
1. 检查环境依赖
2. 提取样本数据
3. 构建数据集
4. 训练模型
5. 进行预测

### 单独运行各个模式

#### 1. 环境检查
```bash
python run_system.py --mode env
```

#### 2. 解压数据文件
```bash
python run_system.py --mode extract --input "2025/01/2025-01-02.7z"
```

#### 3. 构建数据集
```bash
python run_system.py --mode build --input "data/extracted/000001.csv" --code 000001 --date 2025-01-02 --preclose 10.0
```

#### 4. 训练模型
```bash
python run_system.py --mode train --input data/processed/dataset.csv
```

#### 5. 进行预测
```bash
python run_system.py --mode predict --input data/processed/dataset.csv
```

## 直接使用main.py

### 构建模式
处理单个股票CSV文件，提取特征和生成标签：

```bash
python main.py --mode build \
  --input "data/extracted/000001.csv" \
  --code 000001 \
  --date 2025-01-02 \
  --preclose 10.0 \
  --output data/processed/dataset.csv
```

### 解压模式
解压7z压缩的交易日数据：

```bash
python main.py --mode extract \
  --input "2025/01/2025-01-02.7z" \
  --output data/extracted
```

### 训练模式
使用数据集训练LightGBM模型：

```bash
python main.py --mode train \
  --input data/processed/dataset.csv \
  --model-path models/lgbm_model.pkl \
  --train-ratio 0.7 \
  --valid-ratio 0.15
```

### 预测模式
使用训练好的模型进行预测：

```bash
python main.py --mode predict \
  --input data/processed/dataset.csv \
  --model-path models/lgbm_model.pkl \
  --threshold 0.7 \
  --output data/processed/predictions.csv
```

## 参数说明

### 通用参数
- `--mode`: 运行模式 (build/extract/train/predict)
- `--input`: 输入文件路径
- `--output`: 输出文件路径

### 构建模式参数
- `--code`: 股票代码
- `--date`: 交易日期
- `--preclose`: 昨收价

### 训练模式参数
- `--model-path`: 模型保存路径 (默认: models/lgbm_model.pkl)
- `--train-ratio`: 训练集比例 (默认: 0.7)
- `--valid-ratio`: 验证集比例 (默认: 0.15)

### 预测模式参数
- `--model-path`: 模型加载路径 (默认: models/lgbm_model.pkl)
- `--threshold`: 预测概率阈值 (默认: 0.7)

## 输出文件

### 数据集文件
- `data/processed/dataset.csv`: 训练数据集
- `data/processed/dataset_predictions.csv`: 预测结果

### 模型文件
- `models/lgbm_model.pkl`: 训练好的LightGBM模型

### 临时文件
- `data/extracted/`: 解压后的CSV文件

## 常见问题

### 1. 模块导入错误
确保在项目根目录运行命令，或者将项目根目录添加到PYTHONPATH。

### 2. 依赖包缺失
```bash
pip install -r requirements.txt
```

### 3. 数据文件路径错误
使用绝对路径或相对于项目根目录的路径。

### 4. 内存不足
减少处理的数据量，或分批处理。

## 性能优化

### 对于大规模数据
1. 使用项目标准 CSV（`utf-8-sig` 编码，`code` 按字符串读取）
2. 分批处理数据
3. 调整LightGBM参数
4. 使用特征选择

### 加速训练
1. 增加n_jobs参数
2. 调整learning_rate
3. 使用GPU版本的LightGBM

## 结果分析

### 模型评估指标
- ROC-AUC: 模型区分能力
- PR-AUC: 精确率-召回率曲线下面积
- Top-K命中率: 最高概率K个样本的命中率
- 特征重要性: 各特征对预测的贡献

### 预测结果
- limit_up_probability: 涨停概率 (0-1)
- prediction: 是否预测涨停 (0/1)

## 扩展功能

### 批量处理多只股票
可以编写脚本遍历多个CSV文件并合并结果。

### 构建2025年全年数据集

使用专门的批量构建脚本：

```bash
# 构建完整的2025年数据集（可能需要较长时间）
python build_2025_dataset.py --mode full --year 2025 --output data/processed/2025_full_dataset.csv

# 测试模式：只处理前5个文件
python build_2025_dataset.py --mode full --year 2025 --max-files 5 --output data/processed/2025_test_dataset.csv

# 采样模式：每3个文件处理1个
python build_2025_dataset.py --mode full --year 2025 --sample 3 --output data/processed/2025_sample_dataset.csv

# 增量构建模式（推荐用于大数据集）
python build_2025_dataset.py --mode incremental --year 2025 --batch-size 10

# 合并增量数据集
python build_2025_dataset.py --mode merge --output data/processed/2025_full_dataset.csv
```

### 多日数据训练
处理多个交易日的数据并合并用于模型训练。

### 实时预测
集成到实时交易系统中进行在线预测。
