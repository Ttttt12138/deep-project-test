# 日期范围数据集划分功能实现总结

## 实现概述

成功实现了按交易日范围划分数据集的功能，支持两种划分模式：
1. **原有比例模式**：按70/15/15等比例划分（保持向后兼容）
2. **新增日期范围模式**：按具体日期范围划分（符合真实交易逻辑）

## 核心修改文件

### 1. `src/data_processing/dataset_split.py` - 核心划分模块

#### 新增函数：
- `split_dataset_by_date_range()`: 按日期范围划分数据集
  - 支持指定训练集、验证集、测试集的具体日期范围
  - 自动跳过不存在的日期并发出警告
  - 自动处理缺失日期问题

- `_check_missing_dates()`: 检查并警告缺失日期
  - 比较期望日期范围和实际日期范围
  - 提供详细的警告信息

#### 增强函数：
- `split_dataset_by_trading_day()`: 增强支持日期范围参数
  - 添加可选参数：`train_date_range`, `val_date_range`, `test_date_range`
  - 自动检测模式：优先使用日期范围划分，否则使用比例划分
  - 完全向后兼容

- `validate_split()`: 增强验证功能
  - 添加 `expected_ranges` 参数支持日期范围验证
  - 验证日期范围是否符合预期
  - 增强统计信息输出

### 2. `scripts/dataset_builder.py` - 数据集构建器

#### 修改方法：
- `_split_dataset()`: 添加日期范围参数支持
  - 新增参数：`train_date_range`, `val_date_range`, `test_date_range`
  - 模式检测：自动识别日期范围模式或比例模式
  - 保持欠采样逻辑不变

- `process_single_day_two_step()`: 支持日期范围参数
  - 新增参数：`train_date_range`, `val_date_range`, `test_date_range`
  - 参数传递给 `step2_process_and_cleanup()`

#### 新增命令行参数：

## 关键特性

### 1. 自动模式检测
```python
# 自动检测使用哪种模式
use_date_range_mode = (train_date_range is not None or
                      val_date_range is not None or
                      test_date_range is not None)

if use_date_range_mode:
    # 使用日期范围划分
    return split_dataset_by_date_range(df, train_date_range, val_date_range, test_date_range)
else:
    # 使用原有比例划分
    return split_dataset_by_trading_day(df, train_ratio, val_ratio, test_ratio)
```

### 2. 缺失日期处理
自动跳过不存在的日期并发出警告：
```
警告: 训练集缺少以下日期: ['2025-01-11', '2025-01-12', '2025-01-13', '2025-01-14', '2025-01-15']
  期望范围: 2025-01-01 ~ 2025-01-15
  实际范围: 2025-01-01 ~ 2025-01-10
```

### 3. 增强验证功能
```python
# 验证日期范围是否符合预期
expected_ranges = {
    'train': ('2025-01-01', '2025-01-05'),
    'val': ('2025-01-06', '2025-01-06'),
    'test': ('2025-01-07', '2025-01-07')
}
validation_result = validate_split(train_df, val_df, test_df, expected_ranges=expected_ranges)
```

## 使用示例

### 比例模式（原有功能）
```bash
python scripts/dataset_builder.py \
    --input "2025/1/2025-01-02.7z" \
    --split \
    --train-ratio 0.7 \
    --val-ratio 0.15
```

### 日期范围模式（新功能）
```bash
python scripts/dataset_builder.py \
    --input "2025/1/2025-01-02.7z" \
    --split \
    --train-date-start 2025-01-01 \
    --train-date-end 2025-01-05 \
    --val-date-start 2025-01-06 \
    --val-date-end 2025-01-06 \
    --test-date-start 2025-01-07 \
    --test-date-end 2025-01-07
```

### 批量构建使用日期范围
```bash
python build_2025_dataset.py \
    --mode full \
    --split-dataset \
    --train-date-start 2025-01-01 \
    --train-date-end 2025-01-05 \
    --val-date-start 2025-01-06 \
    --val-date-end 2025-01-06 \
    --test-date-start 2025-01-07 \
    --test-date-end 2025-01-07
```

### 程序化调用
```python
from src.data_processing.dataset_split import split_dataset_by_trading_day

# 使用日期范围划分
train_df, val_df, test_df = split_dataset_by_trading_day(
    df,
    train_date_range=('2025-01-01', '2025-01-05'),
    val_date_range=('2025-01-06', '2025-01-06'),
    test_date_range=('2025-01-07', '2025-01-07')
)

# 使用原有比例划分（向后兼容）

    df,
    train_ratio=0.7,
    val_ratio=0.15,
    test_ratio=0.15
)
```
