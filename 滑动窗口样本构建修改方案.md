# 滑动窗口样本构建协议 - 修改方案

## 1. 问题诊断

### 当前实现存在的问题

**❌ 当前实现（点对点模式）**
- 特征提取：每个样本只包含单个Tick的瞬时特征
- 样本定义：样本数 = 总Tick数
- 采样逻辑：基于单点`dist_to_limit`进行分层筛选
- 标签生成：虽然技术上是预测第11个Tick，但特征仍然基于单点

**✅ 目标实现（窗对点模式）**
- 特征提取：每个样本包含连续10个Tick的平铺特征
- 样本定义：样本数 = 每只股票(总Tick数 - 10)
- 采样逻辑：基于窗口末端`dist_to_limit_last`进行分层筛选
- 标签生成：使用窗口后第1个Tick的标签（第11个Tick）

---

## 2. 核心修改策略

### 2.1 样本定义重构

**修改前（点对点）：**
```
样本 i: [Tick_i的特征] → 预测 Tick_{i+1} 是否涨停
```

**修改后（窗对点）：**
```
样本 i: [Tick_i, Tick_{i+1}, ..., Tick_{i+9}] → 预测 Tick_{i+10} 是否涨停
```

### 2.2 特征展开方案

**方案选择：平铺特征（Flattening）**

将10个Tick的核心特征平铺到一行，保留时间序列信息：

```
窗口特征向量：
[
  dist_to_limit_T1, dist_to_limit_T2, ..., dist_to_limit_T10,
  bid_depth_T1, bid_depth_T2, ..., bid_depth_T10,
  order_imbalance_T1, ..., order_imbalance_T10,
  # ... 其他核心特征
]
```

**优势：**
- 保留完整时间序列信息
- 模型可以学习序列模式
- 符合LightGBM输入格式要求

## 6. 技术细节

### 6.1 样本计数公式

**修改前：**
```python
total_samples = len(merged_df)  # 直接统计Tick数
```

**修改后：**
```python
# 按股票统计，每只股票的样本数 = Tick数 - window_size
total_samples = sum(
    len(stock_df) - window_size
    for code, stock_df in df.groupby('code')
)
```

### 6.2 标签对齐逻辑

**修改前：**
```python
# 可能存在对齐问题
df = generate_next_tick_limit_up_label(df, lookahead_ticks=10)
window_df = window_extractor.extract_multi_stock_window_features(merged_df)
```

**修改后：**
```python
# 先生成标签，确保对齐
df = generate_next_tick_limit_up_label(df, lookahead_ticks=10)
# 再构建窗口样本，自动对齐第11个Tick的标签
window_samples = window_builder.build_window_samples(df)
```

### 6.3 采样特征列名

**修改前：**
```python
dist_col = 'dist_to_limit'  # 单点特征
```

**修改后：**
```python
dist_col = 'dist_to_limit_last'  # 窗口末端特征
```

## 10. 总结

本修改方案的核心目标是将当前的"点对点"样本构建模式升级为"窗对点"模式，通过滑动窗口技术将时间序列信息整合到单个样本中，从而为模型提供更丰富的上下文信息。

主要改进：
1. ✅ 样本定义从单点升级为10点窗口
2. ✅ 特征维度从15维扩展到150维（平铺）
3. ✅ 采样逻辑从单点改为窗口末端
4. ✅ 样本计数考虑窗口损耗
5. ✅ 保留完整时间序列信息

预期收益：
- 模型可以学习时间序列模式
- 提升预测准确性
- 减少样本稀释问题
- 更符合实际交易场景
