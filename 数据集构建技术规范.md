# 1. 数据集目标

构建用于训练涨停预测模型的标准化数据集。

## 1.1 模型预测目标

```
预测下一条 Tick 是否涨停
```

## 1.2 标签定义

```python
label = 1 if current(t+1) >= limit_price else 0
```

# 6. 数据清理规范

## 6.1 清理规则

| 清理项       | 条件                      | 处理方式 |
| ------------ | ------------------------- | -------- |
| 无效价格     | current ≤ 0               | 删除     |
| 空盘口       | a1_p ≤ 0 且 b1_p ≤ 0      | 删除     |
| 集合竞价     | time < 09:30:00           | 删除     |
| 重复tick     | time重复                  | 删除重复 |
| 非法盘口     | a1_p < b1_p               | 删除     |
| 成交异常     | volume < 0 或 money < 0   | 删除     |
| 异常跳价     | abs(return) > 10%         | 删除     |

## 6.2 清理后数据要求

```python
current > 0
a1_p >= b1_p
volume >= 0
money >= 0
```

# 7. 涨停价计算

## 7.1 计算公式

```python
limit_price = round(preclose * limit_ratio, 2)
```

## 7.2 涨停比例

| 股票类型 | 涨停比例 | 代码特征        |
| -------- | -------- | --------------- |
| 普通股   | 10%      | 非ST，非30开头  |
| ST股     | 5%       | 代码包含"ST"    |
| 创业板   | 20%      | 30开头          |
| 科创板   | 20%      | 688开头         |
| 北交所   | 30%      | 8开头或4开头    |

## 7.3 数据集要求

数据集中必须包含 `limit_price` 字段。

# 8. 特征构建规范

## 8.1 涨停接近度（4个特征）

| 特征名         | 计算公式                          | 含义           |
| -------------- | --------------------------------- | -------------- |
| dist_to_limit  | (limit_price - current) / limit_price | 距离涨停的百分比 |
| ticks_to_limit | (limit_price - current) / tick_size   | 距离涨停的tick数 |
| ask1_to_limit  | (limit_price - a1_p) / limit_price    | 卖一价距离涨停   |
| ask1_gap       | (a1_p - current) / current            | 当前价到卖一价   |

## 8.2 盘口强度（5个特征）

| 特征名          | 计算公式                                        | 含义           |
| --------------- | ----------------------------------------------- | -------------- |
| bid_depth       | b1_v + b2_v + b3_v + b4_v + b5_v                | 买方总深度     |
| ask_depth       | a1_v + a2_v + a3_v + a4_v + a5_v                | 卖方总深度     |
| order_imbalance | (bid_depth - ask_depth) / (bid_depth + ask_depth) | 买卖不平衡度   |
| b1_volume       | b1_v                                            | 买一量         |
| a1_volume       | a1_v                                            | 卖一量         |

## 8.3 盘口结构（3个特征）

| 特征名     | 计算公式    | 含义           |
| ---------- | ----------- | -------------- |
| spread     | a1_p - b1_p | 买卖价差       |
| ask_slope  | a1_v - a5_v | 卖方斜率       |
| bid_slope  | b1_v - b5_v | 买方斜率       |

## 8.4 成交动量（3个特征）

| 特征名      | 计算公式                        | 含义           |
| ----------- | ------------------------------- | -------------- |
| ret_1tick   | current / current.shift(1) - 1  | 单tick收益率   |
| vol_delta   | volume.diff()                   | 成交量变化     |
| money_delta | money.diff()                    | 成交额变化     |

# 9. 标签构建规范

## 9.1 标签定义

```python
df["label"] = (df["current"].shift(-1) >= df["limit_price"]).astype(int)
```

**含义**：当前tick预测下一tick是否涨停。

## 9.2 删除最后一行

最后一行没有下一tick数据，必须删除：

```python
df = df.iloc[:-1]
```

# 13. 数据集划分规范

## 13.1 划分原则

**必须按交易日切分**，禁止随机切分。

## 13.2 推荐比例

| 数据集 | 比例  | 用途       |
| ------ | ----- | ---------- |
| 训练集 | 70%   | 模型训练   |
| 验证集 | 15%   | 超参调优   |
| 测试集 | 15%   | 最终评估   |

## 13.3 划分示例

```
训练集: 2025-01 至 2025-08
验证集: 2025-09
测试集: 2025-10
```

## 13.4 禁止行为

```
❌ 随机tick切分（导致数据泄漏）
❌ 同一交易日跨集合（导致数据泄漏）
❌ 按股票切分（破坏时间连续性）
```

```python
# 完整的数据处理流程
def process_stock(df: pd.DataFrame, code: str, date: str) -> pd.DataFrame:
    """处理单只股票数据"""
    # 1. 时间解析
    df["time"] = pd.to_datetime(df["time"].astype(str), format="%Y%m%d%H%M%S")
    
    # 2. 时间排序
    df = df.sort_values("time").reset_index(drop=True)
    
    # 3. 数据清理
    df = clean_data(df)
    
    # 4. 计算涨停价
    df["limit_price"] = calculate_limit_price(df["preclose"].iloc[0], code)
    
    # 5. 构建特征
    df = extract_features(df)
    
    # 6. 构建标签
    df = generate_label(df)
    
    # 7. 添加元信息
    df["code"] = code
    df["date"] = date
    
    return df
```
